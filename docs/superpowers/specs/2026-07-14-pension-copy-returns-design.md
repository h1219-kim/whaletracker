# 연기금 따라 투자 수익률 — 설계

날짜: 2026-07-14
상태: 설계 확정 (사용자 승인 대기)

## 1. 목표

"연기금(KRX '연기금등') 순매수 상위 종목을 따라 샀다면 지금 수익률이 얼마일까?"를
두 가지 방식으로 계산해, 지수(KOSPI/코스닥) 대비 성과와 함께 보여준다.

국민연금 종목별 보유는 연 1회 공시라 실시간 추적이 불가능하므로, 매일 갱신되는
연기금 **순매수 흐름**을 "따라 투자" 전략으로 재구성한다.

## 2. 핵심 제약과 해법 (사용자 지적 반영)

- **우리 데이터는 보유가 아니라 매매 흐름(순매수)이다.** → "따라 투자"를 아래처럼 명시적으로 정의한다.
- **연기금 포트폴리오는 종목이 너무 많다.** → **순매수 대금 상위 5개 종목**만 대상으로 한다.
- **연기금은 첫날부터 매도 종목이 있지만 따라 하는 사람은 그 종목이 없다.** →
  **롱온리(long-only)**: 보유하지 않은 종목의 순매도는 **무시**, 보유한 종목은 **보유 수량 한도 내**에서만 매도(공매도 없음).

## 3. 데이터 소스 (2026-07-14 전부 검증 완료)

| 데이터 | 소스 | 형식 | 비고 |
|---|---|---|---|
| 연기금 종목별 순매수 (기간 합계) | KRX `MDCSTAT02401_OUT` (invstTpCd=6000) | JSON | 과거 임의 기간 조회 가능 (검증됨) |
| 연기금 종목별 **일별** 순매수 | KRX 개별종목 투자자별 일별 통계 | JSON | 연속 방식용. 엔드포인트/파라미터는 구현 계획에서 확정 (대안: 대상 5종목의 T0~오늘 일별을 `MDCSTAT02401_OUT` strtDd=endDd로 순회 조회) |

네이버 요청: `User-Agent`, `Referer: https://finance.naver.com`. 요청 간 0.3초 지연.

## 4. 방법론

### 4.1 대상 종목 선정 (두 방식 공통)
- 기간 P ∈ {1개월, 3개월, 6개월}. 시작일 **T0 = 오늘 − P** (거래일 기준 근사).
- **T0 직전 1개월** `[T0−1M, T0]`의 연기금 순매수 상위 **5개** (시장별: 코스피/코스닥 각각).
  - **T0 이전 정보만** 사용 → look-ahead(미래를 미리 아는) 편향 제거 = 공정한 백테스트.
- 비중 `w_i = 종목 i 순매수 대금 / 5개 순매수 대금 합` (5개 안에서 정규화).

### 4.2 스냅샷 추종 — "그때 담아서 그대로 보유"
- T0 종가 `P_i(T0)`에 비중 `w_i`로 매수. 초기 투자금 1로 정규화 → 종목 i 보유주수 `s_i = w_i / P_i(T0)`.
- 각 거래일 t의 포트폴리오 가치 `V(t) = Σ s_i · P_i(t)`.
- 누적수익률 곡선 `R_snap(t) = V(t) − 1`. (리밸런싱 없음, buy-and-hold)

### 4.3 연속 추종 — "매일 연기금 매매를 그대로 따라"
- 대상 5개 종목(T0 시점 상위 5개로 고정)에 대해 T0부터 매일:
  - `f_i(t)` = 종목 i의 그날 연기금 순매수 대금.
  - `f_i(t) > 0`(순매수): 그날 종가로 매수 → `s_i += f_i(t)/P_i(t)`, 현금유입 `+f_i(t)`.
  - `f_i(t) < 0`(순매도): 매도주수 `= min(보유 s_i, |f_i(t)|/P_i(t))`; `s_i −= 매도주수`, 현금회수 `+매도주수·P_i(t)`. **보유 0이면 무시.**
- **시간가중수익률(TWR)** 로 곡선을 만든다 (자본 유입/유출 효과 제거 — "돈을 더 넣어 커진 것"과 "종목이 올라 커진 것"을 분리):
  - 그날 순현금흐름 `CF(t)` = 매수대금 − 매도회수 (당일 종가 기준).
  - 주식가치 `V(t) = Σ s_i(t) · P_i(t)`.
  - 일간수익률 `r(t) = (V(t) − CF(t)) / V(t−1) − 1` (V(t−1)=0인 최초 유입일은 r=0).
  - 누적 `R_cont(t) = Π(1 + r(τ)) − 1`.

### 4.4 벤치마크
- 지수 종가로 `R_idx(t) = Idx(t)/Idx(T0) − 1`.
- 요약에 **초과수익(alpha)** = `R_방식(끝) − R_idx(끝)` 강조.

## 6. API
- `GET /api/returns` → returns.json 패스스루 (없으면 `{"empty": true}`).
- 정적 빌드: `build_static`이 `data/returns.json`을 site로 복사, `build_meta` 신선도 대상에 추가(기대주기 매일=4일).

## 7. UI (연기금 수급 섹션 아래 새 카드 "연기금 따라 샀다면")
- **3개 누적수익률 곡선** 라인차트: 스냅샷 / 연속 / 지수. x=날짜, y=%. 0 기준선.
- **호버**: 크로스헤어 + 툴팁 — 그 시점 세 값 + **두 방식 차이 설명**
  ("스냅샷 = 시작일에 사서 그대로 보유 / 연속 = 매일 연기금 매매를 따라감").
- 토글: 기간(1/3/6개월), 시장(코스피/코스닥). (종목 수 3/5 토글은 구현 때 가볍게 판단)
- 요약 카드: 각 방식 최종 수익률·초과수익, 대상 5종목 리스트(이름·비중).
- dataviz 규칙 준수: 얇은 2px 선, 3시리즈 범례, 크로스헤어 호버, 텍스트는 텍스트 토큰, 테이블 대체 뷰.

## 8. 한계 (화면 하단 고지)
- 순매수 대금을 그날 **종가로 근사**(실제 체결가 아님).
- **순매수 상위 ≠ 연기금 실제 보유**. 상위 5개만 → 전체 포트폴리오 아님.
- 거래비용·세금·배당 **무시**.
- '연기금등'에 국민연금 외 연기금 포함 · 위탁운용분 제외([[nps-data-semantics]]).
- 연속 방식은 대상 5개를 T0에 고정 → 이후 새로 편입되는 종목 미반영.

## 9. 구현 위치
- `nps_fetcher/prices.py` — 네이버 종목/지수 일별 종가 수집·파싱.
- `nps_fetcher/returns.py` — 대상 선정 + 두 방식 계산 → returns.json.
- `nps_fetcher/pipeline.py` — returns 단계 추가(주가 수집이 무거우니 실패해도 나머지 유지).
- `app.py` — `/api/returns`.
- `build_static.py` — returns.json 포함.
- `templates/index.html` + `static/app.js` + `static/style.css` — 새 섹션·라인차트 렌더러.
